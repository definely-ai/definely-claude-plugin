# Contributing to definely-claude-plugin

Thank you for your interest in contributing. This guide covers how to submit changes.

## Before you start

- Open an issue first for any significant change — new skills, behaviour changes, or breaking modifications — so we can align before you invest time writing code.
- For small fixes (typos, broken links, minor clarifications) you can open a PR directly.

## Getting started

1. Fork the repository and create a branch from `main`:
   ```bash
   git checkout -b your-feature-name
   ```
2. Make your changes.
3. Open a pull request against `main`.

## What we accept

| Type | Welcome? |
|---|---|
| New skill (`SKILL.md`) | Yes — open an issue first |
| Improvements to existing skills | Yes |
| Bug fixes | Yes |
| Documentation improvements | Yes |

## Pull request checklist

- [ ] Branch is up to date with `main`
- [ ] Skill files follow the existing `SKILL.md` structure
- [ ] README updated if you added or changed a skill
- [ ] No secrets, API keys, or user data committed

## Code owners

All PRs require approval from a code owner before merging. See [`.github/CODEOWNERS`](.github/CODEOWNERS).

## Questions

Open an issue or contact the team at [support@definely.com](mailto:support@definely.com).
